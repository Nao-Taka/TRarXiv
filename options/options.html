ストレージ設定

設定の保存場所を選択します。ローカルはこのデバイスのみ、同期はChromeアカウントで複数デバイスに同期されます。
同期を使う場合、暗号化されたAPIキーが他デバイスにも同期されます。キーは暗号化済みのため漏洩リスクは下げられますが、完全に安全とは言えません。

ローカルサーバーのセットアップ

Ollama（推奨）

Ollama は macOS / Linux / Windows 対応の無料ローカルLLMランナーです。 OpenAI互換APIを提供します。

1
Ollama をインストール
ollama.com/download からOSに合ったインストーラーをダウンロードして実行してください。
2
モデルをダウンロード
ターミナルで以下のコマンドを実行してください（例: Llama 3.2 3B）: 
ollama pull llama3.2
他のおすすめモデル: gemma2:9b, mistral, phi4-mini
3
Ollama を起動
Ollama アプリを起動するか、ターミナルで: 
ollama serve
起動後、http://localhost:11434 でAPIが利用可能になります。
4
TrArXiv を設定
「LLM設定」→「ローカル」タブで以下を設定してください：
・サーバーURL: http://localhost:11434/v1
・モデル名: ダウンロードしたモデル名（例: llama3.2）
・APIキー: ollama（そのまま）

LM Studio

1
LM Studio をインストール
lmstudio.ai からダウンロードしてインストールします。
2
モデルを選択してサーバーを起動
LM Studio の左パネルで「Local Server」を開き、モデルを選択して 「Start Server」をクリックします。
3
TrArXiv を設定
「LLM設定」→「ローカル」タブで以下を設定:
・サーバーURL: http://localhost:1234/v1
・モデル名: LM Studioで選択したモデル名
・APIキー: lm-studio または任意の文字列

注意事項

Chrome拡張からローカルサーバーへのアクセスには、 サーバー側でCORSヘッダーの許可が必要です。

Ollamaの場合、環境変数を設定してください: 
OLLAMA_ORIGINS=chrome-extension://* ollama serve
macOSのLaunchAgentを使っている場合は、 /etc/launchd.conf または systemd サービスに Environment=OLLAMA_ORIGINS=chrome-extension://* を追加してください。

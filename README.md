# TRarXiv

ArXiv をはじめとする論文サイトの読解を支援する Chrome 拡張機能（Manifest V3）。  
段落単位の翻訳・平易な解説・図解析・論文ブリーフィング・LLM チャットを提供します。

---

## 機能一覧

### 翻訳・解説

| 機能 | 説明 |
|---|---|
| **セクション翻訳** | 各見出し横の「翻訳」ボタンで、段落ごとに原文直下へ和訳を表示 |
| **文対応ハイライト** | 翻訳ブロック内の英文・和文ペアにマウスオーバーすると対応行が強調表示 |
| **原文の表示切り替え** | 翻訳ブロックのボタンで段落ごとに原文を折り畳み／展開 |
| **文字サイズ設定** | 翻訳ブロックの文字サイズを 小・中・大・特大 から選択（設定画面） |
| **平易な解説** | 「解説」ボタンで専門用語を噛み砕いた日本語解説を生成 |
| **全文一括翻訳** | 本文・引用文献・付録をカテゴリ単位でまとめて翻訳 |
| **翻訳進捗表示** | ボタンに「3/8段落 • 残り~12s」を表示しながら翻訳 |

### 論文分析

| 機能 | 説明 |
|---|---|
| **論文ブリーフィング** | 論文タイトル直下の「⚡ 論文ブリーフィング」で概要・手法・新規性・課題を即時要約 |
| **図の解析** | 各図に表示される「図を解析」ボタンでグラフ・表・図の内容を日本語で解説（Vision API） |
| **著者リサーチ** | 著者名横の「🔍 調べる」ボタンで研究分野・代表論文・所属などを表示 |
| **論文ライブラリ** | 開いた ArXiv 論文を自動登録。タグ付け・サマリー管理が設定画面から可能 |

### 拡張サイト対応

| 機能 | 説明 |
|---|---|
| **サイト解析** | ArXiv 以外の論文サイトをポップアップから解析。CSS セレクターを LLM で自動推定 |
| **設定の同期** | サイト設定は `chrome.storage.sync` に保存。同じ Chrome アカウントの全デバイスで共有 |
| **解析結果の修正** | ポップアップのチャット入力でフィードバックを送り LLM にセレクターを再調整させる |
| **ドメイン一括対応** | Nature.com を解析すれば、同ドメインの他論文にも自動適用 |

### チャット・管理

| 機能 | 説明 |
|---|---|
| **論文チャット** | 拡張アイコンをクリックして論文コンテキスト付き LLM チャットを開く |
| **LRU キャッシュ** | 翻訳・解説を `chrome.storage.local` にキャッシュ。容量超過時は古いものから自動削除 |
| **タスク別モデル選択** | 翻訳・解説・チャットでプロバイダー／モデルを個別に指定 |
| **トークン上限** | 1論文あたりのトークン上限を設定。超過前に確認ダイアログを表示 |
| **使用量・コスト記録** | モデルごとの使用トークン数と推定コスト（USD）を自動集計 |
| **APIキー暗号化** | キーを PBKDF2 + AES-256-GCM で暗号化保存。利用時にパスワードで解除 |

---

## 対応サイト

| サイト | 対応状況 |
|---|---|
| ArXiv HTML（`arxiv.org/html/…`） | ✅ 組み込み対応 |
| Nature / Science / PubMed など | ✅ ポップアップから「サイトを解析」で対応追加 |
| その他の論文サイト | ✅ 同上（DOM 構造があれば対応可能） |

> ArXiv は PDF版（`arxiv.org/abs/…` や `arxiv.org/pdf/…`）は対象外です。HTML 版を利用してください。

## インストール

1. Chrome で `chrome://extensions` を開く
2. 右上の **「デベロッパーモード」** をオンにする
3. **「パッケージ化されていない拡張機能を読み込む」** をクリック
4. クローンした `trarxiv/` フォルダを選択

拡張機能一覧に **TRarXiv** が表示されれば完了です。

## 初期設定

### ① APIキーを登録する

拡張アイコンを**右クリック → 「オプション」** を開き、使いたいプロバイダーのタブで APIキーを入力して「保存」します。

| プロバイダー | 取得先 | APIキー形式 |
|---|---|---|
| **OpenAI** | platform.openai.com | `sk-…` |
| **Anthropic** | console.anthropic.com | `sk-ant-…` |
| **Google Gemini** | aistudio.google.com | `AIza…` |
| **OpenRouter** | openrouter.ai | `sk-or-…` |
| **ローカル（Ollama 等）** | — | `ollama`（デフォルト） |

> **セキュリティ**: 「パスワードで暗号化する」を有効にすると PBKDF2+AES-256-GCM でキーを暗号化して保存します。利用時にポップアップでパスワード入力が求められます。

### ② タスク別にモデルを設定する

設定ページ最上部の **「タスク別モデル設定」** カードで翻訳・解説・チャットのモデルを選択します。

**推奨設定の例：**

| タスク | 推奨モデル | 理由 |
|---|---|---|
| 🔤 翻訳 | `gpt-4.1-mini` / `gemini-2.0-flash` | 構造が単純、軽量モデルで十分 |
| 📖 解説 | `gpt-4.1` / `claude-sonnet-4-6` | 背景知識の補足に高性能が必要 |
| 💬 チャット | `gpt-4.1` / `claude-opus-4-7` | 複雑な質問への回答精度を重視 |

## 使い方

### ArXiv 論文を読む

1. `arxiv.org/html/…` の論文ページを開く
2. 各セクション見出し横に **「翻訳」「解説」** ボタンが表示される
3. 論文タイトル直下に **「⚡ 論文ブリーフィング」** ボタンが表示される
4. 各図のキャプション横に **「図を解析」** ボタンが表示される
5. 著者名横に **「🔍 調べる」** ボタンが表示される

### ArXiv 以外の論文サイトに対応させる

1. 対応させたい論文ページ（例: `nature.com/articles/…`）を開く
2. 拡張アイコンをクリック → **「🔍 このサイトを解析」** ボタンをクリック
3. LLM が DOM 構造を解析してセレクターを自動推定・保存
4. 同じドメインの他ページを開くと翻訳ボタンが表示される

**ボタンが表示されない場合**: ポップアップの「⚠ アクセスを許可」ボタンをクリックしてサイトへのアクセス権を付与してください。  
**セレクターを修正したい場合**: ポップアップの「✏ 解析結果を修正」エリアに問題を日本語で入力すると LLM が再調整します。

### チャット

1. `arxiv.org/html/…` の論文ページを開いた状態で拡張アイコンをクリック
2. 論文のタイトル・アブスト・セクション情報が自動でコンテキストとして渡される
3. 「この論文の手法は何ですか？」「○○ とはどういう意味ですか？」などを自由に質問

## ローカルサーバー (Ollama)

### インストール・起動

```bash
# Ollama インストール後
OLLAMA_ORIGINS=chrome-extension://* ollama serve

# モデルのダウンロード
ollama pull llama3.2       # 軽量・高速（翻訳向け）
ollama pull gemma2:9b      # バランス型（解説・チャット向け）
```

**macOS LaunchAgent / Linux systemd で常時起動する場合は `OLLAMA_ORIGINS=chrome-extension://*` を環境変数に設定してください。**

### TRarXiv の設定

オプション画面の **「ローカル」** タブで：
- サーバーURL: `http://localhost:11434/v1`
- APIキー: `ollama`
- 「接続テスト」でモデル一覧を自動取得 → タスク別モデル設定で割り当て

## よくある質問

**Q. ArXiv のページを開いてもボタンが表示されない**  
A. URL が `arxiv.org/html/` で始まっているか確認してください。PDF ページや論文一覧ページには対応していません。

**Q. Nature などのサイトで解析したのにボタンが表示されない**  
A. ポップアップに「⚠ アクセスを許可」バナーが表示されている場合は「許可する」をクリックしてください。許可後にページをリロードすると翻訳ボタンが表示されます。

**Q. 翻訳ボタンを押してもエラーになる**  
A. オプション画面でタスク別モデル設定が保存されているか、および該当プロバイダーの APIキーが入力されているか確認してください。

**Q. APIキーのパスワード入力が繰り返し求められる**  
A. Chrome を完全に閉じると復号済みキーがクリアされます（`chrome.storage.session` の仕様）。再起動後の最初の操作時に一度パスワードを入力すれば、同一セッション中は再入力不要です。

**Q. キャッシュをクリアしたい**  
A. オプション画面の **「使用量・コスト → キャッシュ」** タブから削除できます。

**Q. 翻訳済みのセクションを再翻訳したい**  
A. 「再翻訳」ボタンをクリックするとキャッシュを無視して再翻訳します。

**Q. トークン上限を設定したい**  
A. オプション画面の **「LLM設定 → トークン上限」** で有効にして上限値を設定してください。

**Q. 翻訳の文字が小さい**  
A. オプション画面の **「表示設定 → 翻訳文字サイズ」** で「大」または「特大」を選択してください。
